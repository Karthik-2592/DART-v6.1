Query-Based Routes

The general process of a query based Route Handler should be as follows:
1. With the help of the query string, find the corresponding ID
2. Use ID based (deterministic) Route Handlers to then perform the required tasks.

This requires that ID resolver functions to be present which resolve query string to its corresponding ID.
Furthermore split the index.js into componenet .js files (each associated with their endpoint route handling)

===========================
User Query-Based Routes
===========================
GET    /users?username=:username             → Get user info by username
GET    /users?email=:email                   → Get user info by email
GET    /users/search?q=:query                → Search users by keyword (username, display_name, description)

PUT    /users?username=:username             → Bulk modify user info (request body includes new values; unspecified fields retain original values)

===========================
Song Query-Based Routes
===========================
GET    /songs?title=:title                   → Get song(s) by exact title
GET    /songs/search?q=:query                → Search songs by keyword (title, genre, release_year)

PUT    /songs?title=:title                   → Bulk modify song metadata (request body includes new values; unspecified fields retain original values)

===========================
Contributor Query-Based Routes
===========================
POST   /songs?title=:title/contributors      → Add contributor(s) to a song by title
GET    /songs?title=:title/contributors      → List contributors for a song by title
GET    /users?username=:username/contributions → List songs contributed by a user by username

===========================
Favorites Query-Based Routes
===========================
POST   /favorites?username=:username&title=:title → Favorite a song by username + song title
DELETE /favorites?username=:username&title=:title → Unfavorite a song
GET    /favorites?username=:username&title=:title/playcount → Get play count for a user’s favorite song
POST   /favorites?username=:username&title=:title/play      → Increment play count for a user’s favorite song

===========================
User Favorites Query-Based Routes
===========================
GET    /users/:username/favorites            → Get all songs favorited by a user

===========================
Playlist Query-Based Routes
===========================
POST   /playlists?creator=:username          → Create playlist for a user by username
GET    /playlists?name=:name&creator=:username → Get playlist details by name + creator username
PUT    /playlists?name=:name&creator=:username → Bulk modify playlist metadata (request body includes new values; unspecified fields retain original values)
DELETE /playlists?name=:name&creator=:username → Delete playlist by name + creator username

POST   /playlists?name=:name&creator=:username/songs → Add song(s) to playlist by name
DELETE /playlists?name=:name&creator=:username/songs/:title → Remove song by title from playlist
GET    /playlists?name=:name&creator=:username/songs → List songs in playlist

POST   /playlists?name=:name&creator=:username/share/:username → Share playlist with user by username
DELETE /playlists?name=:name&creator=:username/share/:username → Unshare playlist with user by username
GET    /playlists?name=:name&creator=:username/shares → List users playlist is shared with

===========================
Song Play Count Query-Based Routes
===========================
POST   /songs?title=:title/play              → Increment global play count by song title
GET    /songs?title=:title/playcount         → Get global play count by song title
GET    /songs/top?genre=:genre               → Get top songs sorted by play count (filter by genre)

===========================
Follower/Following Query-Based Routes
===========================
POST   /users/:username/follow/:targetUsername → Follow another user by username
DELETE /users/:username/follow/:targetUsername → Unfollow user by username
GET    /users/:username/followers              → List followers of a user by username
GET    /users/:username/following              → List users followed by a user by username
